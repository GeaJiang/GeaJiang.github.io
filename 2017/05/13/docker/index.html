Posted on 2017-05-13 | In backend

docker 安装

我本来是想选择 docker toolbox ,希望用一套纯命令行工具来操作docker，但是却似乎要淘汰掉了，加之总是出问题，还是选择了一键安装的docker。

在mac上安装毫无压力，一路点下去就好。win系统需要专业版及企业版，因为家庭版的内核有些功能没有开启，不能使用docker。

Dockerfile的构建

官网上的文档写的很清晰，只不过有些经验，需要看书或者其他教程来获得。

Dockerfile首先从 From 开始，设置一个基础镜像，按照我走过的经验来看，以一个 node:latest 作为基础镜像，使用起来还是有难度的，启动直接进入node环境，可以以 node index.js 来启动项目。我个人比较喜欢从一个系统镜像开始，一步一步的构建下去，但是这样镜像会大一些，似乎不是一个很好的选择。

From 一个基础系统镜像的话建议选择最新的镜像，尽管说我们构建镜像是希望保持与当前的生产环境相同，但是你做好镜像以后这个镜像可以运用到生产环境了，而且在构建系统环境的时候使用，yum install 或者 apt-get install 某些安装包的时候，陈旧的版本可能让你找不到你所需要的安装包，譬如一些图形编译 libjpeg8-dev 等。

对于一些Dockerfile中命令，官方的解释在我看来还算清楚，这里讲一下书中介绍的一些经验。首先，RUN WORKDIR COPY 这些命令都会产生一层结构，对于 RUN 这个命令尽量能将需要运行的命令合并起来，然后构建镜像的时候构建成一层。

Dockerfile不支持交互的模式，所以所有命令都应该是一键的，这就要求我们在使用 yum install 或者 apt-get install 都要加上 -y 这个参数，然后我个人遇到的问题是，在安装npm包时，需要登陆安装几个私有包，在Dockerfile不支持交互的前提下，我们不能在构建时去登陆，npm官网上有介绍如何使用在Dockerfile中使用需要登陆的私有包，我最终的解决方法是把登陆过的.npmrc文件放在项目中使用 COPY 进入镜像，运行 npm install 或者 yarn 就可以了。
